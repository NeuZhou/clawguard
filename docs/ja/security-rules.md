# セキュリティルールリファレンス

## 組み込みルール

### 1. プロンプトインジェクション検出 (`prompt-injection`)

**OWASP:** LLM01

受信メッセージをスキャンし、インジェクション攻撃を検出します。

| カテゴリ | 検出内容 | 重大度 |
|---------|---------|--------|
| 直接インジェクション | 指示の上書き、ロール変更、脱獄 | critical |
| デリミタインジェクション | チャットテンプレート区切り文字 | critical/high |
| エンコード | Base64、ゼロ幅文字、ホモグリフ | high |
| 間接 | 隠し指示、HTMLコメント | high/warning |
| マルチターン | 偽コンテキスト参照 | warning |

### 2. データ漏洩検出 (`data-leakage`)

**OWASP:** LLM06

送信メッセージをスキャンし、APIキー・認証情報・個人情報を検出。検出時にキーローテーションURLを提供。

### 3. 異常検出 (`anomaly-detection`)

メッセージ連射、トークン爆弾、ループ、コストスパイク、サブエージェントカスケード、無限リトライ、ネットワークフラッドを検出。

### 4. コンプライアンス (`compliance`)

ファイルシステム変更、権限昇格、外部アクセス、ツール呼び出しを追跡。

### 5. ファイル削除保護 (`file-protection`)

**OWASP:** LLM02

破壊的なファイルシステム操作を検出。重要パス（`~`、`/`、`.ssh`、`.env`）対象時はcriticalに昇格。
